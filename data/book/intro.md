# 引言

## 新兴群体：“赤脚程序员”

上世纪六七十年代，在中国的农村地区，活跃着这样一群“赤脚医生”。

他们大多只接受过短期的医疗培训，却为广大农民提供了最基础的医疗服务，极大改善了当时农村地区缺医少药的困境。

![](/book/intro/59f3775324f8.png)

而在今天的软件开发领域，也活跃着一个类似的群体。

**他们大多没有计算机科班的背景。**

他们不懂复杂的算法，看到满屏的代码有时也会头晕。

**但他们更懂生活，更懂业务，更懂痛点。**

他们可能是想管好库存的小店主； 

可能是想记录孩子成长的父母； 

也可能是想让工作流程自动化的白领。

![](/book/intro/e44ae5ba3098.png)

过去，借助一些低代码平台简单的“拖拉拽”，他们也能完成一些最基础的应用开发。

但由于不理解代码运行逻辑，往往只能做出一个半成品。遇到复杂问题，还是得求助于专业开发者。

![](/book/intro/b38554d96dc1.png)

放在以前，这样的技术门槛几乎无法绕过。

但随着 AI 编程工具的兴起，这一局面正在悄然发生改变。

那些原本需要掌握专业技能，才能完成的开发任务，已经可以通过“自然语言指令”来实现。

编程不再意味着手写枯燥的代码，而是变成了一种“对AI说人话”的创作方式。

![](/book/intro/50461c91663e.png)

**我们与软件之间的距离，从未如此之近。**

## ShowMeCode 是什么

这**不是**一本传统的编程教材。

我们不教枯燥的计算机理论，也不培养大厂螺丝钉。

这是一本“赤脚程序员”的实战手册。

我们的目标只有一个：**帮你把脑海中的“点子”，变成别人能用的“产品”。**

为此，你需要掌握的不仅仅是“Vibe Coding”，而是：

![](/book/intro/5e9328ad8e1c.png)

1.  **想清楚**：别急着动手。学会像产品经理一样拆解需求，识别伪需求，写出AI能看懂的提示词。

2.  **写出来**：学会指挥 Claude Code 等工具，让它们做你的御用程序员。同时，掌握最基础的工程规范，避免代码变成“屎山”。

3.  **变好看**：这是一个看脸的时代。学会基础的设计规范和审美，别让粗糙的界面劝退用户。

4.  **发出去**：代码写完只是完成了一半。学会购买域名、配置服务器、部署上线，让全世界都能访问你的作品。

我们始终信奉：

**问题来源于生活，解决方案也应掌握在你自己手中。**
